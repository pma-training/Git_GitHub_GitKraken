# What is Git, GitHub and GitKraken
Git, GitHub and GitKraken, and how and why PMA has decided to use this platform (_Français ci-dessous_)

_Video:_ [What is Git, GitHub and GitKraken](https://www.youtube.com/watch?v=yGTF2LAsuWA&list=PLaCCIQf3NY979c70cnegKx8Cmo3Wltkia&index=2&t=0s)

### Summary 
Introduce users to the new PMA data management version control system that is based in Git, GitHub and GitKraken. This video goes through Git, GitHub and GitKraken, what each system does, how PMA uses each system, and why PMA decided to change version control systems. 


#### By the end of this video, you should be able to answer the following questions:
- What is Git?
- How does PMA use Git?
- What is GitHub?
- How does PMA use GitHub?
- What is GitKraken?
- How does PMA use GitKraken?
- Why did PMA decide to start using Git, GitHub, and GitKraken?


### Definitions
**Github** is an open-source program for tracking changes and text files. It is the core technology that GitHub is built on top of. PMA does not use Git directly.

**GitHub** is a website that provides remote access to Git. PMA uses GitHub to organize organizations, host repositories and approve changes to .do files. 

**GitKraken** is a Git GUI client for Windows, Mac, and Linux and it helps developers become more productive and efficient with Git. PMA uses GitKraken to record version history.

### Why the new system?
PMA chose to change to the new system for many reasons, including, but not limited to:
1. Eliminates need to track version history within the .do files themselves, making them easier to read
2. Facilitates tracking of changes, so it is less likely that people will forget to record updates to .do files
3. Makes it easier to identify the exact versions of .do files that were used to develop specific data products 
4. Simplifies synchronization between users making sure that all users have access to the most recent versions of the .do files


### Glossary of terms:
| Term | Definition |
| ---- | ---------- |
| Git | An open source program for tracking changes in text files |
| GitHub | A website that provides remote access to Git. PMA houses its organizations and repositories on GitHub |
| GitKraken | A software for Windows, Mac and Linux that helps developers interact with Git. PMA uses GitKraken to interact with Git and GitHub |
| Organization | A workspace where teams can collaborate across many projects at once. PMA uses organizations to organize .do files by purpose and country. |
| Pull | When changes are taken from the GitHub server to a local copy of the repository. GitKraken pulls data from GitHub to keep the two platforms synced |
| Push | Sending committed changes to a remote version of the repository. GitKraken pushes data to GitHub to keep the two platforms synced |
| Repository | Contains all of the project files and documentation and stores each file’s revision history. Can have multiple collaborators. PMA keeps its .do files in repositories that are organized by survey type and action |
| Tag | Active points to commits that do not move. PMA tags the final commit before releasing data products so the version of the specific .do file can be easily referenced for future use |

# Qu’est-ce que Git, GitHub et GitKraken ?

### Résumé  
Présente aux utilisateurs et utilisatrices le nouveau système de contrôle de versions et de gestion des données de PMA sur Git, GitHub et GitKraken. Cette vidéo passe en revue Git, GitHub et GitKraken, en expliquant ce que fait et comment PMA utilise chaque système, et pourquoi PMA a décidé de changer son système de contrôle de versions.

#### À la fin de cette vidéo, vous devriez connaître les réponses aux questions suivantes :

### Définitions
**Git** est un programme en accès libre permettant le suivi des modifications dans des fichiers de traitement de texte. C’est la technologie sur laquelle se repose GitHub. PMA n’utilise pas Git directement.

**GitHub** est un site internet offrant un accès à distance à Git. PMA utilise GitHub pour organiser ses « organizations », héberger ses repositories et autoriser les modifications des .do files. 

**GitKraken** est un client GUI Git pour Windows, Mac et Linux, et aide les développeurs à devenir plus productifs et efficaces en utilisant Git. PMA utilise GitKraken pour enregistrer l’historique des versions. 


### Pourquoi le nouveau système ?
PMA a choisi de s’orienter sur ce nouveau système pour de nombreuses raisons, entre autres :
1. Éliminer le besoin de suivre l’historique des versions dans les .do files eux-mêmes, les rendant ainsi plus faciles à lire. 
2. Faciliter le suivi des modifications, réduisant ainsi le risque d’oublier de rapporter les mises à jour des .do files. 
3. Faciliter l’identification des versions exactes des .do files qui ont été utilisées pour développer des produits de données spécifiques. 
4. Simplifier la synchronisation entre les utilisateurs et utilisatrices, en s’assurant qu’ils/elles aient tous/toutes accès aux dernières versions des .do files.


### Glossaire :
| Terme | Definition |
| ---- | ---------- |
| Git | Logiciel en accès libre permettant le suivi des modifications dans des fichiers de traitement de texte. |
| GitHub | Site internet offrant un accès remote (à distance) à Git. PMA héberge ses organizations et repositories sur GitHub. |
| GitKraken | Logiciel pour Windows, Mac et Linux aidant les développeurs à interagir avec Git. PMA utilise GitKraken pour interagir avec Git et GitHub. |
| Organization | Espace de travail où les équipes peuvent collaborer sur différents projets en même temps. PMA utilise des organizations pour organiser ses .do files selon leur objectif et par pays |
| Pull | Lorsque les modifications sont « tirées » (extraites) du serveur GitHub vers une copie d’un repository en local. GitKraken tire les données de GitHub pour que les deux plateformes soient synchronisées. |
| Push | Envoyer des modifications d’un commit vers une version remote (à distance) d’un repository. GitKraken push (pousse) les données sur GitHub pour que les deux plateformes soient synchronisées. |
| Repository | Contient tous les fichiers et la documentation du projet, et stocke l’historique de révision de chaque fichier. Peut avoir plusieurs collaborateurs. PMA garde ses .do files dans des repositories organisés par type d’enquête et d’action |
| Tag | Points actifs des commits qui ne bougent pas. PMA attribue un tag au commit final avant de publier les produits de données pour que la version spécifique d’un .do file soit référencée et puisse être facilement utilisée plus tard |
